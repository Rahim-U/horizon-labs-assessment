# Git Repository Setup Guide

This guide will help you initialize and push your repository to GitHub.

## Step 1: Initialize Git Repository

```bash
# Navigate to project root
cd D:\task-management-assessment

# Initialize git repository
git init

# Set default branch to main (if not already set)
git branch -M main
```

## Step 2: Add All Files

```bash
# Add all files to staging
git add .

# Verify what will be committed
git status
```

## Step 3: Create Initial Commit

```bash
# Create initial commit
git commit -m "Initial commit: Task Management Application

- Add FastAPI backend with authentication and task management
- Add React frontend with TypeScript and Tailwind CSS
- Add comprehensive documentation
- Add database migrations
- Add configuration files"
```

## Step 4: Create GitHub Repository

1. Go to GitHub: https://github.com/new
2. Create a new repository:
   - Repository name: `task-management-assessment` (or your preferred name)
   - Description: "Full-stack task management application"
   - Visibility: Choose Public or Private
   - **DO NOT** initialize with README, .gitignore, or license (we already have these)

## Step 5: Connect and Push to GitHub

After creating the repository on GitHub, you'll see instructions. Use these commands:

```bash
# Add remote repository (replace YOUR_USERNAME with your GitHub username)
git remote add origin https://github.com/YOUR_USERNAME/task-management-assessment.git

# Verify remote was added
git remote -v

# Push to GitHub
git push -u origin main
```

## Alternative: Using SSH

If you prefer SSH:

```bash
# Add remote with SSH
git remote add origin git@github.com:YOUR_USERNAME/task-management-assessment.git

# Push to GitHub
git push -u origin main
```

## Verify

After pushing, verify on GitHub:
- All files are present
- README.md displays correctly
- Both frontend and backend folders are visible

## Future Commits

For future changes:

```bash
# Stage changes
git add .

# Commit with descriptive message
git commit -m "Description of changes"

# Push to GitHub
git push
```

## Important Notes

- The `.gitignore` file is already configured to exclude:
  - `node_modules/` (frontend)
  - `venv/` (backend)
  - `*.db`, `*.db-shm`, `*.db-wal` (database files)
  - `.env` files (environment variables)
  - Build outputs (`dist/`, `build/`)
  - Cache files (`__pycache__/`, `.cache/`)

- Never commit:
  - `.env` files with secrets
  - Database files
  - `node_modules/` or `venv/`
  - Personal API keys or tokens
